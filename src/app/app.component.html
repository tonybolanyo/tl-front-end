Basic

  • Design the component using CSS and HTML.
  • Make it work with the Array<Day> generated in the AppComponent (days)
  • Add an input attribute [multiple]="boolean" to the component. If multiple true, the component will allow selecting more than one date, in other case, only once.

Medium

  • Add pagination to the component and the buttons to control it. Look at the shadows that appears in both sides if there are more pagination.
  • Make it compatible with the reactive form.
  • Do test cases you consider necessary in order to check the component works properly.
  • Run lint and fix lint issues.

Advanced

  • Make it Responsive: Ensure that the component adapts to its parent container when the browser is resized or the parent container size decreases. It's important not to reset pagination and to handle resizing in a way that the page adjusts as if it were initially loaded. For example:
    • Initial window: [1,2,3,4,5]
    • Click next: [6,7,8,9,10]
    • When resizing to a width where only 3 elements fit, the displayed elements should be [7,8,9]. This is because if you start from the beginning, it would be [1,2,3], [4,5,6], [7,8,9].

Finally, generate a git patch with all changes and send it to sergio@tulotero.es.

If you encounter any difficulties while working on the exercise, please feel free to send the patch even if you aren't able to complete any of the latest point. We encourage you to submit your progress or any code changes you've made, even if it's not the final solution.
